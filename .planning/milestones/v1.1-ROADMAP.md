# Milestone v1.1: Social Profile Metadata + Card Refresh

**Status:** ✅ SHIPPED 2026-03-10
**Phases:** 7-12 plus inserted Phase 08.1
**Total Plans:** 22

## Overview

v1.1 turned supported social links into profile-oriented surfaces by persisting social identity metadata, separating profile-authored bios from generic page descriptions, refreshing simple and rich cards around avatar-first identity cues, and extending the experience with public follower-history analytics plus card-level sharing.

## Milestone Summary

**Decimal Phases:**

- Phase 08.1: Custom Profile Descriptions (inserted after Phase 8 so the description contract/capture work could land before the broader docs and regression pass)

**Key Accomplishments:**

- Social profile metadata now persists handles, profile imagery, audience counts, and profile-authored descriptions through public and authenticated paths.
- Simple and rich cards now render supported social links as avatar-first profile surfaces while preserving stable fallback cards for unsupported links.
- Rich profile cards can optionally show a separate full-width description-image row when preview media is distinct from the profile image.
- OpenLinks now publishes append-only public follower-history CSVs plus a lazy-loaded analytics surface reachable from the profile header and cards.
- Card-level share actions now sit beside analytics, and maintainer docs now route routine CRUD toward AI workflows or Studio before manual JSON editing.

**Key Decisions:**

- Social profile metadata remained optional and additive so unsupported links and manual JSON workflows would not break.
- Audience stats stayed build-time only; no client-side live platform API fetching was introduced.
- `profileDescription` became a distinct field from generic `description` so supported profile cards could prefer the profile-authored bio without disturbing non-profile description behavior.
- Rich-card follow-up work extended the shared shell and policy layers instead of forking platform-specific component trees.
- Public follower-history assets were published as append-only per-platform CSVs rather than one shared multi-column history table.

**Issues Resolved During Milestone:**

- Normalized metadata merge semantics for manual versus generated profile fields.
- Card analytics visibility and placement gaps discovered in UAT.
- Share payload and visibility gaps discovered in UAT.
- Docs discoverability gap around the preferred CRUD path.

**Issues Deferred:**

- Broader social audience extraction coverage beyond the currently supported/first-pass platforms.
- Broader template branching and alternate card anatomies beyond the shared profile/fallback shell.
- Further Studio expansion beyond the currently documented self-serve/editor scope.

**Technical Debt Accepted:**

- `bun run quality:check` still fails the `/` total bundle-byte budgets for both mobile and desktop.
- SEO still warns that social preview metadata is using the deterministic fallback image.
- The analytics chart chunk remains large and should stay code-split or be reduced in a future cleanup pass.
