### Gap

- The only live referral example in the real repo still relies on generated referral output instead of a source-authored `links[].referral` object. `cluborange-referral` is present in [data/links.json](/Users/peterryszkiewicz/.codex/worktrees/a075/open-links/data/links.json), but it is still authored as a generic rich link with `profileSemantics: "non_profile"` and gets its referral disclosures only through generated artifacts. That leaves the milestone’s intended maintainer-authored manual-first flow unproven in the live source data.

## Broken / Incomplete Flow

### Maintainer-authored referral link flow

Intended flow:

1. Maintainer authors a referral link directly in `data/links.json`
2. Enrichment/merge fills blanks only where useful
3. Runtime renders referral disclosure UI
4. Verification commands confirm the final result

Current break:

- Step 1 is not represented by any real referral-bearing link in `data/links.json`

Impact:

- The repo proves generated enrichment -> render
- The repo does not yet prove source-authored manual referral -> enrich/merge -> render on real project data

## Tech Debt / Residual Risks

### Phase 19

- Generated referral augmentation only applies to rich links right now, which is reasonable technically but still worth making explicit in future cleanup or docs clarification.
- Generated referral artifacts expose completeness and source URLs, but not per-field provenance.

### Phase 20 / 21

- Studio is still an Advanced JSON fallback for referral editing rather than a first-class CRUD experience.

### Shared repo debt

- `bun run quality:check` currently passes performance and accessibility, but still warns that the social preview image uses the deterministic fallback image.
- LinkedIn authenticated cache remains stale.
- Rumble public enrichment remains partial for follower count.

## Recommendation

This milestone should be treated as **gaps_found**, not because the implementation is broadly broken, but because one critical end-to-end maintainer flow is still only implied by docs/tests rather than exercised by real source data.

Best next step:

1. Add at least one explicit `links[].referral` object to a real link in `data/links.json`
2. Re-run:
   - `bun run enrich:rich:strict`
   - `bun run validate:data`
   - `bun run build`
3. Re-audit the milestone

If you accept that gap as tolerable debt, the remaining issues are non-blocking and mostly documentation/ops polish rather than feature incompleteness.
